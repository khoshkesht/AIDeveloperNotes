<b>عامل فراتر از Model</b>

ساختن Agent قابل‌اعتماد فقط به انتخاب Model وابسته نیست. در این نگاه، Agent از ترکیب Model و Harness شکل می‌گیرد؛ Model تنها بخش کوچکی از معادله است و سهم اصلی به Harness مربوط می‌شود. Harness همان لایه‌ای است که Sandbox، Tool، Orchestration و Guardrails را کنار هم قرار می‌دهد تا سیستم Agentic Coding بتواند قابل‌اتکا کار کند. بدون این لایه، خروجی Model به‌تنهایی برای ساخت یک سیستم پایدار کافی نیست.

<b>Verification Loop انسانی و خودکار</b>

Verification Loop فقط به test خودکار محدود نمی‌شود. یک sub-agent می‌تواند خروجی Agent را بررسی کند، روی آن iterate کند و کیفیت را بالا ببرد؛ اما بعضی شرایط باید human in the loop verification یا review را فعال کنند. داده‌هایی که از این reviewها ساخته می‌شوند، می‌توانند به golden data set تبدیل شوند و مسیر self-improvement را باز کنند. ترکیب automated verification و بررسی انسانی، پایه‌ای برای قابل‌اعتماد شدن autonomous systems در production است.

#Post_14

<b>نقشه سیستم با Open Knowledge Format</b>

Open Knowledge Format با مجموعه‌ای از markdown fileها کار می‌کند که هر کدام نماینده یک چیز در سیستم هستند؛ مثل service، database یا contract. این فایل‌ها به یکدیگر link می‌شوند و هم انسان می‌تواند آن‌ها را بخواند، هم Agent می‌تواند آن‌ها را بخواند و edit کند. ترکیب این ساختار با Graph RAG می‌تواند به Agent کمک کند قبل از نوشتن code، ارتباط entityها را ببیند و اثر تغییر یک بخش را روی کل سیستم بفهمد.

#Post_15

<b>Context Engineering به جای Dump کردن Repository</b>

Context Engineering یعنی Agent فقط با انباشتن کل repository در context window جلو نرود. برای code baseهای بزرگ، representation ساختاریافته از سیستم می‌تواند مفیدتر باشد؛ مخصوصاً وقتی Agent باید بفهمد اگر X تغییر کند، چه بخش‌هایی affected می‌شوند. Graph RAG با دنبال‌کردن connectionها میان markdown fileها، رابطه بین بخش‌های code base را روشن می‌کند. این نگاه به Agent کمک می‌کند مثل engineer یا architect فقط یک file را تغییر ندهد، بلکه impact آن را هم بسنجد.

#Post_16

<b>کاربردهای Long-running Agent</b>

Long-running Agent زمانی ارزشمند است که کار به زمان، iteration و پوشش گسترده context نیاز دارد. Deep Research یک نمونه موفق است، چون answer فقط یک پاسخ کوتاه نیست و معمولاً contextual scaffolding لازم دارد. AI coding هم نمونه مهم دیگری است، چون Agent می‌تواند مرتب code را اجرا کند و مطمئن شود تغییرات جدید مسیر را خراب نمی‌کنند. نکته اصلی این است که اجرای طولانی بدون verification می‌تواند زمان و هزینه را هدر بدهد.
